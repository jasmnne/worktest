Historically, most companies would buy into a single ecosystem for their productivity needs.
Pre-cloud tools to communicate and collaborate on files were typically called the intranet. In broad strokes, there are four main jobs to be done in team collaboration:
Team Communication & Social Network

Then: Confluence, Sharepoint, Lync, Basecamp

Now: Slack, Hipchat, Yammer, Microsoft Teams, Basecamp, Facebook Workplace

Workflow & Project Management

Then: Confluence, Sharepoint, Basecamp

Now: Asana, Jiro, Trello

Cloud Storage & Shared Network Drives

Then: Network Drive, Confluence, Sharepoint, Basecamp

Now: Box.com, Dropbox, Onedrive, Google Drive

Knowledge Centers

Then: Confluence, Sharepoint

Now: Cogo

We moved from a single solution for many different tasks, to cloud tools that do a single job very well and integrate with other services. That's exactly what Cogo aims to be. We are completely focused on enabling the best knowledge center functionality.

Cogo vs. Sharepoint

Cogo provides customizability and robustness without sacrificing usability. It's fast and easy to set up every time, but doesn't cut corners on what it can do.

Learn how Cogo compares to Sharepoint

Cogo vs. Confluence

Cogo is lightweight and elegant, and easy to integrate with other modern collaboration tools. Tagging systems, powerful search functionality, and a built-in presentation mode allow for information to be found, shared, and presented easily.

Learn how Cogo compares to Confluence

Cogo vs. Box.com

Cogo makes information easy to discover by implementing a different type of organization and file sorting mechanism. It makes use of deep folder hierarchies and can be implemented as a fully-featured, front-end service as well.
